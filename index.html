01 / Full Stack Development

Two sides, one product

Full stack development means owning both sides of the web — the interface a person touches and the engine that powers it.

Client-Side

Front-End

Everything the user sees and touches. Built in the browser, rendered on their screen, reacting to their every click.

  • HTML5 — Semantic Document Structure
  • CSS3 / Bootstrap 5.3 — Styling & Layout
  • JavaScript ES6+ — Live Interactivity
  • React / Vue — Component Frameworks
  • Fetch API — Consuming REST endpoints

Runs in: Browser

Server-Side

Back-End

The invisible engine. Processes requests, runs business logic, talks to databases — never seen, always felt.

  • Node.js / Python / PHP — Server Runtimes
  • Express / Django / Laravel — Frameworks
  • REST / GraphQL — API Design
  • MySQL / MongoDB — Data Persistence
  • JWT / OAuth — Auth & Security

Runs on: Server

Structure that flexes

Bootstrap's 12-column grid rearranges these cards as the viewport changes. Hover each to feel the micro-interaction.

HTML5 Semantics

Semantic elements give meaning beyond appearance. They guide screen readers, search crawlers, and browsers turning a visual layout into a structured document.

<header><nav><main> <section><article><aside> <footer>

Bootstrap 12-Col Grid

Bootstrap wraps CSS Flexbox into a named, breakpoint-driven 12-column system. Columns collapse on mobile and expand at higher viewports mobile-first by default.
